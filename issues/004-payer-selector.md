# Выбор плательщика при сохранении чека

## What to build

Добавить UI для выбора кто заплатил за чек на шаге review в обеих страницах ввода (TextEntry и PhotoEntry). Сейчас `payer_id` захардкожен = 1 (Макар), из-за чего долги считаются неправильно.

Нужен переключатель на 2 варианта: **Макар** (user) или **Ксюша** (girlfriend). Выбранное значение передаётся как `payer_id` в `POST /api/receipts`.

## Acceptance criteria

- [ ] На странице TextEntry в шаге review появляется переключатель "Кто заплатил?"
- [ ] На странице PhotoEntry в шаге review появляется переключатель "Кто заплатил?"
- [ ] Переключатель влияет на `payer_id` в запросе к `POST /api/receipts` (1 = Макар, 2 = Ксюша)
- [ ] При сохранении чека с payer_id=2 (Ксюша), долг считается корректно на странице Balance
- [ ] Существующие тесты на сервере проходят
