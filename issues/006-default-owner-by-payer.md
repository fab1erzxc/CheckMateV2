# Умный дефолт owner в зависимости от плательщика

## What to build

Сейчас все распарсенные AI товары получают `owner: 'user'` по умолчанию. Если за чек платила Ксюша, пользователю приходится вручную переключать каждый товар на `'girlfriend'` или `'50-50'`.

Нужно:
1. Если выбран плательщик = Макар → новые товары по умолчанию `owner: 'user'` (как сейчас)
2. Если выбран плательщик = Ксюша → новые товары по умолчанию `owner: 'girlfriend'`
3. Опционально: кнопка "Назначить всё на X" в шапке ParsedItemsTable, где X — текущий плательщик

## Acceptance criteria

- [ ] При выборе плательщика = Ксюша, все новые товары получают `owner: 'girlfriend'`
- [ ] При выборе плательщика = Макар, все новые товары получают `owner: 'user'` (поведение не меняется)
- [ ] После AI-парсинга товары наследуют owner от выбранного плательщика
- [ ] Пользователь всё ещё может переключать owner индивидуально через OwnerToggle
- [ ] Кнопка "Set all to [name]" в ParsedItemsTable меняет owner всех товаров разом
